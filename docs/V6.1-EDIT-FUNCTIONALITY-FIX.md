# V6.1 - Edit Functionality Fix for Embed Page

## Issue

When clicking "Edit" on a dive log in the embed page, the journal would open but the "Update Dive Entry" button would not update or close the journal upon saving.

## Root Cause Analysis

1. **State Mismatch**: The embed page was using `editLogIndex`/`setEditLogIndex` but the components expected `editingLog`/`setEditingLog`
2. **Prop Mismatch**: The embed page was passing `userId` instead of `nickname` to `DiveJournalSidebarCard`
3. **Handler Chain Broken**: The edit callback chain wasn't properly connected between components

## Fixes Applied

### 1. Fixed State Management in embed.jsx

**Before:**

```jsx
const [editLogIndex, setEditLogIndex] = useState(null);
```

**After:**

```jsx
const [editingLog, setEditingLog] = useState(null);
```

### 2. Updated Component Props

**Before:**

```jsx
<DiveJournalSidebarCard
  userId={userId}
  editLogIndex={editLogIndex}
  setEditLogIndex={setEditLogIndex}
  // ...
/>
```

**After:**

```jsx
<DiveJournalSidebarCard
  nickname={getDisplayName()}
  editingLog={editingLog}
  setEditingLog={setEditingLog}
  // ...
/>
```

### 3. Fixed Sidebar Props and Dependencies

**Before:**

```jsx
handleEdit: () => {}, // Add if needed
editLogIndex,
```

**After:**

```jsx
handleEdit: (log) => setEditingLog(log), // Set the log to edit
editingLog,
```

### 4. Updated useMemo Dependencies

**Before:**

```jsx
}, [
  // ... other deps
  editLogIndex,
]);
```

**After:**

```jsx
}, [
  // ... other deps
  editingLog,
]);
```

### 5. Fixed Journal Close Handler

**Before:**

```jsx
setEditLogIndex(null);
```

**After:**

```jsx
setEditingLog(null);
```

## How Edit Functionality Now Works

1. **User clicks Edit**: Calls `handleEdit(log)` which sets `editingLog` state
2. **Journal Opens**: `DiveJournalDisplay` receives `editingLog` prop
3. **Form Pre-fills**: `useEffect` detects `editingLog` and:
   - Sets `isEditMode = true`
   - Switches to `activeTab = "add-new"`
   - Pre-fills form with existing log data
4. **Submit Button**: Shows "Update Dive Entry" instead of "Save Dive Entry"
5. **Update Logic**: `handleSubmit` preserves existing `log.id` when `isEditMode = true`
6. **Save & Close**: After successful save:
   - `setIsDiveJournalOpen(false)` closes journal
   - `setEditingLog(null)` clears edit state
   - Storage events trigger sidebar refresh

## Component Flow

```
embed.jsx
  └── DiveJournalSidebarCard (receives editingLog/setEditingLog)
      └── DiveJournalDisplay (receives editingLog prop)
          └── Edit Button onClick → setEditingLog(log) → triggers edit mode
          └── Submit Button → handleSubmit → update logic → close journal
```
